ONDE ESTAMOS
─────────────────────────────────────────

O BrainIAc Engine está funcional como produto completo:

• Extração de legendas de qualquer canal do YouTube (vídeos, shorts, lives,
  podcasts, cursos, playlists) com delta inteligente
• Agente RAG local com BM25 + Ollama — sem API key, sem custo, offline
• Provedores externos opcionais via BYOK (Gemini, OpenAI, Claude)
• Streaming de respostas, histórico de conversa, multi-canal
• Histórico de extrações anteriores com acesso rápido por canal
• Notificação nativa do sistema ao concluir extração
• Interface web responsiva em PT/EN/ES
• Sync opcional com Google Drive (OAuth2, escopo drive.file)
• Distribuição via Electron com instalador Windows (NSIS)
• Auto-update via GitHub Releases
• Copyright © 2026 CriAugu — CNPJ 65.131.075/0001-57

   → Rodar scripts/setup-env.ps1 (Python + pip + yt-dlp + node)
   → npm run build no web_interface

   → Publicar .exe no GitHub Releases

PRÓXIMO SPRINT — PRODUTO E NEGÓCIO
─────────────────────────────────────────

#3  Telemetria opt-in (CEO + Investidor)
    Solução definida: Posthog (gratuito até 1M eventos/mês)
    → Modal de consentimento na primeira abertura
    → Eventos: extração iniciada/concluída, provedor IA usado, canal indexado,
      chat enviado, erros críticos
    → Zero dado pessoal, zero conteúdo das transcrições
    → Pendente: criar conta no Posthog e implementar SDK

#4  ICP definido + página de vendas (CEO)
    Ação: entrevistar 5 pessoas antes de escrever uma linha de código
    Perfis a entrevistar: pesquisador acadêmico, jornalista investigativo,
    criador de conteúdo, analista de inteligência de mercado, educador
    Página de pré-venda: Gumroad ou Lemon Squeezy (30 minutos)
    → Descobre se alguém paga ANTES de construir o sistema de licença

#5  Sistema de licença (CEO)
    Solução definida: Lemon Squeezy (venda + chave de ativação + webhooks)
    → Na primeira abertura do app: campo para inserir chave de licença
    → Validação: chamada HTTP à API do Lemon Squeezy
    → Pendente: conta no Lemon Squeezy + implementar validação no Electron
    → Pré-requisito: ICP e página de vendas (#4) antes de implementar

#7  Dependência yt-dlp — Plano B (Investidor)
    Decisão: aceitar o risco no curto prazo
    Justificativa: yt-dlp tem 15 anos de histórico, comunidade ativa, atualiza
    em horas quando o YouTube muda algo. Risco gerenciável.
    Plano B futuro: Whisper local (transcrição de áudio sem legenda)
    → Feature "Extrair mesmo sem legenda" para v2.0
    → Requer download do áudio (~50MB por hora de vídeo) + modelo Whisper
    → Custo: ~3-5min de processamento por hora de vídeo em CPU

─────────────────────────────────────────
EVOLUÇÃO DO PRODUTO
─────────────────────────────────────────

Fase 1 — concluída
  Pipeline local → Drive → NotebookLM (manual)
  Agente RAG próprio com Ollama (sem dependência externa)
  Histórico de canais, notificações, streaming de respostas

Fase 2 — próxima (pós-distribuição)
  Telemetria opt-in (Posthog)
  Sistema de licença (Lemon Squeezy)
  ICP validado com 5 entrevistas
  Página de pré-venda ativa

Fase 3 — médio prazo
  Modelos Ollama alternativos configuráveis pelo usuário
  Automação da importação no NotebookLM via API (quando disponível)
  Servidor MCP — base RAG acessível por qualquer agente Claude/GPT
  Whisper como fallback para vídeos sem legenda

Fase 4 — longo prazo
  Modo institucional: BrainIAc em servidor interno, múltiplos usuários
  API pública para integração com outras ferramentas
  Suporte a fontes além do YouTube (podcasts RSS, PDFs locais)

─────────────────────────────────────────
PRINCÍPIO QUE NÃO MUDA
─────────────────────────────────────────

A extração via yt-dlp tem que continuar local. É o coração da proposta.
Qualquer SaaS centralizado que fizesse isso em escala seria derrubado pelo
YouTube em semanas. A execução distribuída na máquina de cada usuário é a
proteção natural — e o diferencial competitivo real do BrainIAc.
