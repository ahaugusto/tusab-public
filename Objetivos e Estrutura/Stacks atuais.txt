─────────────────────────────────────────
BACKEND
─────────────────────────────────────────

Python 3.11 (embeddable no build de produção)
FastAPI — servidor web e API REST (localhost:8001)
Uvicorn — servidor ASGI
yt-dlp — extração de legendas e metadados do YouTube (binário bundled)
Pandas — manipulação do banco de dados CSV
Google Auth / Google API Python Client — OAuth2 e Drive (escopo drive.file)

─────────────────────────────────────────
AGENTE RAG
─────────────────────────────────────────

rank_bm25 — busca lexical BM25Okapi com cache em memória por canal
  → Sem embeddings, sem API externa para indexação
  → Enriquecimento: tags do YouTube (peso 3×) + keywords TF-IDF (peso 2×) + descrições do vídeo
  → Query expansion: LLM gera 2 variações da pergunta, scores combinados por média
  → Multi-canal: busca simultânea em múltiplos índices com merge e re-ranking

Ollama — LLM local padrão (llama3.2:1b, ~1.3 GB)
  → Zero custo, zero API key, funciona offline
  → Auto-instalado na primeira execução via download do GitHub
  → Serve via HTTP em localhost:11434

Provedores externos opcionais (BYOK — Bring Your Own Key):
  OpenAI SDK — chat (gpt-4o-mini)
  Anthropic SDK — chat (claude-sonnet-4-6)
  Google Generative AI SDK — chat (gemini-1.5-flash)

Streaming de respostas — fetch + ReadableStream, cursor piscante na UI
Histórico de conversa — últimas 6 mensagens incluídas no contexto do LLM

─────────────────────────────────────────
ESTRUTURA DE DADOS EM PRODUÇÃO
─────────────────────────────────────────

Arquivos do app  → %ProgramFiles%\BrainIAc\
Dados do usuário → %AppData%\BrainIAc\data\
  cerebro_txt\   — transcrições .txt extraídas por canal
                   (cabeçalho com atribuição ao criador original)
  gestao\        — CSVs de metadados e histórico de extração
  agent_index\   — índices BM25 em JSON por canal ({canal}_index.json)
  temp\          — VTTs temporários durante extração (auto-removidos)
  config\        — agent_config.json (provider/chave) e token.json (OAuth Drive)

─────────────────────────────────────────
PADRÕES E ARQUITETURA
─────────────────────────────────────────

RAG local — BM25 + query expansion + LLM local (Ollama) sem dependência de nuvem
Local-first — todos os dados ficam na máquina do usuário; Drive é opcional
BYOK — usuário traz própria chave de API para provedores externos
OAuth2 Desktop Flow — autenticação Drive sem servidor próprio (escopo drive.file)
Streaming SSE — respostas do agente via ReadableStream para UX em tempo real
Threading com Events — controle de pausa/cancelamento sem matar processos
Multi-canal — índices independentes por canal, busca simultânea com merge
Camada de atribuição — toda transcrição carrega crédito ao criador original
ELECTRON_RUN / BRAINIAC_DATA_DIR — vars de ambiente para isolar dev/produção
Tema adaptativo — detecta tema do OS, persiste preferência do usuário

─────────────────────────────────────────
PRINCÍPIO DA STACK
─────────────────────────────────────────

Stack deliberadamente local e agnóstica de provedor. Sem banco de dados externo,
sem autenticação própria, sem infraestrutura de servidor — tudo roda na máquina
do usuário. O agente RAG mantém esse princípio: índice BM25 em JSON local,
LLM via Ollama sem API key. O usuário que quiser mais qualidade conecta sua
própria chave Gemini/GPT/Claude — mas não é obrigado.

A distribuição via Electron resolve o empacotamento para usuário leigo: Python,
Node.js, yt-dlp e Ollama são todos gerenciados automaticamente — o usuário
clica duas vezes e usa, sem configurar ambiente.
