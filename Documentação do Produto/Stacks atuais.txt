─────────────────────────────────────────
BACKEND
─────────────────────────────────────────

Python 3.12 (embeddable no build de produção)
FastAPI — servidor web e API REST (localhost:8001)
Uvicorn — servidor ASGI
yt-dlp — extração de legendas e metadados do YouTube (binário bundled)
Pandas — manipulação do banco de dados CSV
Google Auth / Google API Python Client — OAuth2 e Drive (escopo drive.file)
pdfplumber — extração de texto de PDFs
python-docx — extração de texto de DOCX
python-multipart — upload de arquivos via FastAPI

Dependências opcionais (imagens e áudio):
  pytesseract — OCR fallback para imagens (requer Tesseract binário: winget install tesseract)
  faster-whisper — transcrição local de áudio, modelo base ~150 MB, roda em CPU sem GPU

─────────────────────────────────────────
AGENTE RAG
─────────────────────────────────────────

rank_bm25 — busca lexical BM25Okapi com cache em memória por canal
  → Imagens indexadas como documentos (descrição ou OCR convertida para .txt)
  → Áudios indexados como documentos (transcrição convertida para .txt)
  → Enriquecimento: tags YouTube (3×) + keywords TF-IDF (2×) + descrições
  → Query expansion: LLM gera variações da pergunta, scores combinados
  → Multi-canal: busca simultânea em múltiplos índices com merge
  → Anti-alucinação: threshold BM25 + verificação pós-geração por keyword overlap

Ollama — LLM local padrão (llama3.2:1b, ~1.3 GB)
  → Zero custo, zero API key, funciona offline
  → Auto-instalado na primeira execução via download do GitHub

Provedores externos opcionais (BYOK):
  Groq — llama-3.1-8b-instant (rápido) e llama-3.1-70b-versatile (qualidade)
  OpenAI SDK — gpt-4o-mini
  Anthropic SDK — claude-sonnet-4-6
  Google Generative AI SDK — gemini-1.5-flash

Streaming de respostas — fetch + ReadableStream, cursor piscante na UI
Histórico de conversa — últimas 6 mensagens no contexto do LLM

FRONTEND — ESTRUTURA MODULAR
─────────────────────────────────────────

React 19 + Vite + Tailwind CSS 3 + Framer Motion + Lucide React
react-i18next — internacionalização PT / EN / ES

Organização modular (desde v0.4.1):

  src/
    constants/index.js          ← API_BASE, LANGS, BTN_FOCUS
    services/api.js             ← 24 funções de API centralizadas
    hooks/
      useStatus.js              ← polling /status (2s)
      useAgentStatus.js         ← polling /agent/status (3s)
    components/
      home/HomeScreen.jsx       ← tela inicial com cards
      chat/ChatDrawer.jsx       ← drawer lateral de chat
      sidebar/SidebarContent.jsx← menu lateral recolhível
      agent/
        OllamaSetup.jsx         ← status e download do modelo
        RepositorioTab.jsx      ← gestão da base de conhecimento
        RelatorioTab.jsx        ← relatório de extrações por canal
      extraction/
        ExtractionModal.jsx     ← seletor de tipos de conteúdo
        PostExtractionModal.jsx ← modal pós-extração
      shared/
        ModalWrapper.jsx        ← backdrop único: focus trap, Escape, aria-dialog
        GuideModal.jsx          ← guia de uso (6 passos)
        Onboarding.jsx          ← wizard de primeiro acesso
        StatCard.jsx            ← card de métrica reutilizável
        LogLine.jsx             ← linha de log com severity
    App.jsx                     ← orquestração: estado + handlers + layout

Padrão de comentários DevOps aplicado em todos os arquivos:
  · @file, @description, @module, @author, @copyright
  · @param, @returns em componentes e hooks
  · Seções marcadas: // ─── Nome ───────────
  · Handlers documentados com JSDoc inline

Responsividade:
  Breakpoints: sm (640px) · md (768px) · lg (1024px)
  · Mobile (<768px): nav drawer deslizante, logo compacta, botões full-width
  · Tablet (md): nav rail lateral fixa, painel logo na Home, sem drawer
  · Desktop (lg): logo 320px, padding ampliado, layout split na Home

Acessibilidade (WCAG 2.1 AA):
  · Touch targets ≥ 44×44px em todos os botões interativos
  · aria-label em todos os botões de ícone sem texto visível
  · aria-expanded em painéis expansíveis
  · role="dialog" aria-modal="true" via ModalWrapper em todos os modais
  · Focus trap + Escape-to-close em modais
  · caption sr-only em tabelas de dados
  · aria-live="polite" no status de extração

PADRÕES E ARQUITETURA

PRINCÍPIO DA STACK
─────────────────────────────────────────

Stack deliberadamente local e agnóstica de provedor.
A extração via yt-dlp roda no IP residencial do usuário — proteção natural
contra bloqueios do YouTube e diferencial competitivo inimitável em SaaS.

O frontend é modular e documentado para facilitar manutenção e evolução.
O backend é simples: FastAPI + arquivos locais, zero banco de dados externo.
