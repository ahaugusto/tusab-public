─────────────────────────────────────────
ONDE ESTAMOS (v0.4.0)
─────────────────────────────────────────

O BrainIAc evoluiu de "ferramenta de YouTube" para
sistema de gestão de conhecimento pessoal multi-fonte:

• Base de conhecimento: YouTube + PDF + DOCX + MD + TXT + texto colado
• Repositório com gestão via interface (adicionar, visualizar, deletar)
• Relatório de extração por canal com stats e tabela de vídeos
• Home screen com cards de navegação contextual
• Chat flutuante acessível de qualquer aba
• Provedor Groq (llama-3.1-70b gratuito) como alternativa ao Ollama
• Agente RAG local com BM25 + camada de anti-alucinação
• Extração de qualquer canal YouTube com atribuição ao criador
• 4 abas: Extração · Repositório · Relatório · Configurar Agente

─────────────────────────────────────────
SPRINT IMEDIATO — DISTRIBUIÇÃO
─────────────────────────────────────────

1. Publicar OAuth no Google Cloud Console
   → Testing → Production (privacy policy e TOS prontos)
   → Desbloquia Drive para qualquer usuário Google

2. Registro INPI
   → inpi.gov.br > Programa de Computador (~R$90)
   → Titularidade: CriAugu CNPJ 65.131.075/0001-57

3. Gerar primeiro instalador oficial
   → setup-env.ps1 → npm run build web_interface → npm run build electron
   → Publicar no GitHub Releases para auto-update

4. Primeira venda B2B
   → Demo com criador educacional (50k–500k inscritos)
   → Case da AUVP como referência
   → Mostrar base multi-fonte: canal + documentos do criador

─────────────────────────────────────────
PRÓXIMO SPRINT — PRODUTO
─────────────────────────────────────────

#1  Imagens na base (v2)
    → OCR com Tesseract ou visão com LLaVA via Ollama
    → Abre: fotos de slides, capturas de tela, infográficos
    → Estimativa: médio esforço

#2  Sistema de licença (Lemon Squeezy)
    → Tela de ativação no Electron
    → Validação HTTP simples
    → Pré-requisito: ter tração com vendas antes de implementar

#3  Telemetria opt-in (Posthog)
    → Modal de consentimento na primeira abertura
    → Eventos: fontes adicionadas, perguntas feitas, erros críticos
    → Necessário para iterar com dados reais

#4  Onboarding interativo
    → Guia passo a passo na primeira vez em cada aba
    → Tooltips contextuais nas funcionalidades novas
    → Reduz churn de usuários que não entendem o produto

#5  Servidor MCP
    → Expõe a base RAG como ferramenta para agentes externos
    → Usuário conecta BrainIAc ao Claude Code, Cursor, etc.
    → O BrainIAc vira uma "ferramenta de busca" para qualquer agente

EVOLUÇÃO DO PRODUTO
─────────────────────────────────────────

Fase 1 — concluída
  YouTube → base RAG → chat local (Ollama)
  Multi-fonte: PDF, DOCX, MD, TXT, texto
  Repositório, Relatório, Home com cards

Fase 2 — próxima
  Licença + telemetria + imagens
  Primeira venda B2B formal com contrato

Fase 3 — médio prazo
  Servidor MCP para integração com agentes externos

Fase 4 — longo prazo
  Modo institucional (servidor interno, múltiplos usuários)
  API pública para integrações
  Fontes além do YouTube: podcasts RSS, páginas web, código

─────────────────────────────────────────
MUDANÇA DE PERSPECTIVA (v0.4.0)
─────────────────────────────────────────

Antes: "ferramenta de YouTube" — concorre com SummarizeYT, Glasp
Depois: "PKM com IA local" — concorre com NotebookLM, Notion AI, Obsidian AI

O diferencial que nenhum deles tem:
  · Roda local, sem servidor obrigatório
  · YouTube como fonte nativa com extração automática
  · Custo zero para uso básico
  · Privacidade total — dados do usuário são do usuário

─────────────────────────────────────────
PRINCÍPIO QUE NÃO MUDA
─────────────────────────────────────────

A extração via yt-dlp tem que continuar local.
A execução distribuída na máquina de cada usuário é a proteção natural
contra bloqueios do YouTube — e o diferencial competitivo real do BrainIAc.

Tudo mais pode evoluir. Esse princípio não.
