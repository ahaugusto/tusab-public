Perguntas frequentes

Abra a aba Extração e clique em Adicionar Canal. Cole a URL do canal (formatos aceitos: /@canal, /channel/ID, /c/nome). Escolha um nome de projeto, selecione as fontes e clique em Iniciar. O progresso aparece em tempo real no painel de logs.

Disponível para o perfil Pesquisador. No modal de extração, escolha a opção arXiv em vez de YouTube, digite um tema ou palavras-chave e escolha quantos artigos deseja baixar (até 50). O Tusab baixa os PDFs e prepara o texto para indexação — o restante do fluxo (indexar, conversar) é igual ao de qualquer outro documento. Feature de busca acadêmica inspirada no projeto open-source OpenScience (synthetic-sciences).

O Tusab expõe sua base de conhecimento indexada como um servidor MCP (Model Context Protocol) local — qualquer agente de IA compatível pode consultar suas fontes diretamente, sem sair da ferramenta que você já usa (editor de código, assistente de tarefas, etc). Vá em Admin → MCP Server e clique em Copiar configuração MCP. Cole o conteúdo no arquivo de configuração MCP do seu cliente — por exemplo, .cursor/mcp.json no Cursor, ou o arquivo de configuração MCP do Claude Code. Tudo roda localmente; nenhum dado sai da sua máquina.

Disponível para o perfil Pesquisador. No modal de extração, escolha a opção FHIR em vez de YouTube, digite um tema ou palavras-chave e escolha quantos estudos deseja baixar (até 50). O Tusab busca em um servidor público de padrão internacional de dados de saúde (o mesmo formato usado por sistemas hospitalares), restrito a estudos de pesquisa — nunca dados de paciente, mesmo que fosse só um dado de teste. O texto é preparado para indexação como qualquer outro documento.

Sim, automaticamente — disponível para qualquer perfil, sem precisar ativar nada. Ao fazer upload de um arquivo (.txt, .md, .pdf ou .docx) no Repositório, o Tusab identifica se é uma petição (pelo vocativo ao juízo), um contrato (pelas cláusulas numeradas) ou um parecer (pelo cabeçalho de ementa), e extrai automaticamente campos como partes, cláusulas e CPF/CNPJ antes de indexar — melhorando a qualidade das respostas do chat sobre esse documento. Um aviso confirma quando o formato é detectado. Não depende de nenhum serviço externo.

O Ollama permite executar modelos de linguagem (LLMs) localmente, sem enviar dados para a nuvem. É a opção recomendada para quem valoriza privacidade. Alternativamente, você pode usar OpenAI, Anthropic, Gemini ou Groq configurando uma chave de API na aba Agente.

O Tusab usa RAG (Retrieval-Augmented Generation): ao fazer uma pergunta, o sistema recupera os trechos mais relevantes da base indexada usando BM25. Esses trechos são enviados ao LLM junto com sua pergunta. Ative a Busca Ampla para usar o CrossEncoder e melhorar a qualidade da recuperação (ligeiramente mais lento).

Na aba Repositório, clique em Indexar base, marque os projetos que deseja indexar e confirme. Após a indexação, abra o chat e selecione o projeto no seletor de canal.

Por padrão, não. O Tusab é local-first: a extração, indexação e busca acontecem inteiramente na sua máquina. Dados saem apenas se você (a) configurar um LLM externo (OpenAI, Anthropic, Gemini, Groq) ou (b) ativar o Google Drive. Você escolhe.

Na aba Repositório, abra o projeto desejado e use o botão de upload. São aceitos PDF, DOCX, TXT, CSV, MD. Conversas do WhatsApp (Android e iOS) e transcrições de reunião (Zoom, Teams, Otter) são detectadas automaticamente e estruturadas antes de salvar.

O Tusab verifica atualizações automaticamente ao abrir. Quando uma nova versão estiver disponível, um aviso aparece na landing ou em um banner no topo. Clique em Instalar agora — o app fecha, instala e reabre automaticamente, exibindo uma confirmação com a nova versão.

O Tusab envia notificações nativas do sistema operacional em dois momentos: quando uma extração termina e quando o chat responde enquanto o drawer está fechado ou você está em outra aba.

Para ativar: na aba Admin, na seção "Notificações do sistema", clique em Ativar notificações e permita quando o navegador/Electron solicitar. Também é possível ativar durante o onboarding inicial.

Para desativar: clique no ícone de cadeado ou engrenagem na barra de endereço e altere a permissão de notificações para "Bloquear". No Windows, você pode também gerenciar pelo Painel de Controle → Configurações de Notificações.

Nota: Se as notificações estiverem bloqueadas pelo sistema, o botão na aba Admin exibirá instruções para desbloqueá-las manualmente.

Tusab checks for updates automatically on launch. When a new version is available, a notice appears on the landing screen or as a top banner. Click Install now — the app closes, installs, and reopens automatically showing a confirmation with the new version number.

Tusab sends native OS notifications in two situations: when an extraction finishes and when the chat replies while the drawer is closed or you are on another tab.

To enable: go to the Admin tab, find the "System notifications" section, and click Enable notifications. Allow when prompted by the browser or Electron. You can also enable during the initial onboarding flow.

To disable: click the lock or gear icon in the address bar and set notification permission to "Block". On Windows, you can also manage this in Settings → Notifications.

Note: If notifications are blocked at the system level, the button in the Admin tab will show instructions for manually unblocking them.
