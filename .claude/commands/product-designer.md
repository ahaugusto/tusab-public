Adote o papel descrito em @agents/product-designer.md.

Para a feature, problema ou oportunidade descrita, entregue uma proposta integrada:

1. **Job to be Done**: qual perfil (Estudante/Especialista/Pesquisador), qual trabalho, qual o caminho atual até o valor?
2. **Fricção atual**: fluxo + visual — onde o usuário trava, onde a hierarquia esconde o próximo passo?
3. **Solução**: especificação implementável — mudança de fluxo (passos, microcopy) + mudança visual (tokens, estados, arquivo afetado)
4. **Impacto**: na métrica de ativação (instalar → extrair → chat com fonte) ou no diferencial percebido (▶ MM:SS, local-first, multi-fonte)
5. **Custo**: estimativa de esforço (horas) e risco de regressão

Para diagnóstico de fluxo isolado, use `/ux`.
Para auditoria visual isolada, use `/ui`.

Este agente fecha o ciclo entre os dois.
